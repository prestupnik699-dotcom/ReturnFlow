# PURPOSE

This document records every meaningful change made to the ReturnFlow project.

It serves as the official history of development and ensures transparency for developers, AI assistants, testers, and project owners.

Every released version must have an entry.

Never modify previous changelog entries after release. If a correction is needed, create a new entry.

--------------------------------------------------

# CHANGELOG FORMAT

Each version entry must include:

- Version Number
- Release Date
- Status
- Summary
- New Features
- Improvements
- Bug Fixes
- Database Changes
- API Changes
- Breaking Changes
- Known Issues
- Migration Notes (if applicable)
